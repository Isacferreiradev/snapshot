# SnapDeck.pro

Pay-per-use URL screenshot service. Paste a URL, get HD desktop + mobile screenshots, pay $4.99, download a ZIP. No signup, no subscription, no bullshit.

---

## Prerequisites

- **Node.js** 18 or higher — https://nodejs.org
- **Stripe account** — https://dashboard.stripe.com (free to create)
- **Stripe CLI** (for local webhook testing) — https://stripe.com/docs/stripe-cli

---

## Setup

### 1. Clone and install

```bash
git clone <your-repo-url>
cd snapshot-pro
npm install
```

> Installing Puppeteer will download Chromium (~170 MB). This is normal and required.

---

### 2. Configure environment variables

Copy the example file:

```bash
cp .env.example .env
```

Then edit `.env` and fill in each value:

```env
PORT=3000
BASE_URL=https://snapdeck.pro

STRIPE_SECRET_KEY=sk_test_...
STRIPE_PUBLISHABLE_KEY=pk_test_...
STRIPE_WEBHOOK_SECRET=whsec_...
PRICE_CENTS=499
```

**Where to find your Stripe keys:**

| Variable | Where to find it |
|---|---|
| `STRIPE_SECRET_KEY` | Stripe Dashboard → Developers → API keys → Secret key |
| `STRIPE_PUBLISHABLE_KEY` | Stripe Dashboard → Developers → API keys → Publishable key |
| `STRIPE_WEBHOOK_SECRET` | Generated by the Stripe CLI (see step 3 below) |

> Use **test mode** keys (`sk_test_...`, `pk_test_...`) for development. Never commit your `.env` file.

---

### 3. Set up webhook forwarding (local dev)

In a **separate terminal**, run:

```bash
stripe listen --forward-to snapdeck.pro/api/webhook
```

The CLI will print something like:

```
> Ready! Your webhook signing secret is whsec_abc123...
```

Copy that `whsec_...` value and paste it as `STRIPE_WEBHOOK_SECRET` in your `.env` file.

Keep this terminal running while you test.

---

### 4. Start the server

```bash
npm start
```

Open [https://snapdeck.pro](https://snapdeck.pro).

## How it works

1. User pastes a URL and clicks **Capture Screenshots**
2. Server validates the URL and launches a headless Chromium browser
3. Two screenshots are captured: desktop (1440×900 @2x) and mobile (390×844 iPhone)
4. A watermarked preview PNG is generated and shown in the UI
5. User clicks **Pay $4.99** → redirected to Stripe Checkout
6. Stripe sends a `checkout.session.completed` webhook to `/api/webhook`
7. Server marks the job as paid
8. Stripe redirects user back to `/?success=true&jobId=...`
9. Frontend detects the success params and auto-triggers download after 1.5s
10. Server streams a ZIP containing both full-res PNGs, then deletes the temp files

## Deploying to Railway

Railway is the simplest zero-config deployment for this project.

### Steps

1. Create a free account at [railway.app](https://railway.app)
2. Click **New Project** → **Deploy from GitHub repo** → connect your repo
3. Railway detects Node.js and uses `npm start` automatically
4. In your project settings → **Variables**, add all values from your `.env`:

   | Key | Value |
   |---|---|
   | `PORT` | `3000` (or leave empty — Railway sets this automatically) |
   | `BASE_URL` | Your Railway app URL, e.g. `https://snapshot-pro.up.railway.app` |
   | `STRIPE_SECRET_KEY` | Your live secret key (`sk_live_...`) |
   | `STRIPE_PUBLISHABLE_KEY` | Your live publishable key (`pk_live_...`) |
   | `STRIPE_WEBHOOK_SECRET` | From the Stripe webhook you'll create next |
   | `PRICE_CENTS` | `499` |

5. In the Stripe Dashboard → **Developers → Webhooks** → **Add endpoint**:
   - URL: `https://your-app.up.railway.app/api/webhook`
   - Event: `checkout.session.completed`
   - Copy the signing secret → paste as `STRIPE_WEBHOOK_SECRET` in Railway

6. Redeploy after updating variables.

> **Puppeteer on Railway**: Railway runs Linux containers, so Puppeteer's bundled Chromium works without any extra configuration. If you hit sandbox issues, ensure `--no-sandbox` is in the launch args — it already is in this codebase.

## Troubleshooting

### Webhook not received / payment doesn't trigger download

**Symptom:** Payment succeeds on Stripe, but the page stays on the success screen and the download never starts (404 or 402 from `/api/download`).

**Causes and fixes:**
- The Stripe CLI is not running. Start it: `stripe listen --forward-to snapdeck.pro/api/webhook`
- `STRIPE_WEBHOOK_SECRET` in `.env` doesn't match the secret printed by `stripe listen`. Copy the `whsec_...` value again and restart the server.
- In production: verify the webhook endpoint URL in the Stripe Dashboard matches your deployed URL exactly.

---

### Puppeteer fails with "Running as root without --no-sandbox is not supported"

**Symptom:** Server crashes on first capture with a sandbox error.

**Fix:** This is already handled — `--no-sandbox` and `--disable-setuid-sandbox` are in the Puppeteer launch args. If you still see it, ensure you're running Node 18+ and that `puppeteer` is fully installed (`npm install` completed without errors).

---

### ZIP download is empty or returns 410

**Symptom:** The ZIP downloads but is 0 bytes, or the API returns `410 Gone`.

**Cause:** The screenshot files were already deleted (either from a previous download, or the job expired after 2 hours).

**Fix:** Start a new capture — go back to the homepage and submit a new URL. Jobs are single-use by design.
